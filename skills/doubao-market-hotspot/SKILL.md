# 股民简报 / 市场信息与事件解读 Skill

把用户关注的**市场整体、宏观/政策/新闻/风险事件、跨资产联动和市场情绪**,从公开信息中整理成**对话框市场解读正文**。目标是直接回答「核心判断是什么、事实怎么变、为什么重要、如何传导、分歧在哪里、后续怎么验证」;HTML 只是用户需要展开版来源账本和完整链路时的可选交付物。

## 路由边界

### 触发场景
用户关注的是**市场整体、宏观事件、政策事件、新闻事件、风险事件、资产联动或市场情绪**,而不是某一个具体公司、行业或板块的深度分析。

### 典型触发
- 今天市场为什么涨 / 跌?
- 最近有哪些市场热点?
- 降息对股市有什么影响?
- 地缘冲突对全球资产有什么影响?
- 最近资金风险偏好有什么变化?
- 这条政策新闻对市场意味着什么?
- 欧洲央行加息后,美联储会不会跟着加息?

### 不触发条件
- 某只股票今天为什么涨跌 → 转「个股日报」。
- 某个板块今天为什么涨跌 → 转「板块热度分析」。
- 某个行业长期怎么看 → 转「行业分析」。
- 某家公司基本面如何 → 转「公司分析」。
- 财报 / 业绩数据 → 转「财报与业绩分析」。
- 荐股、目标价、买卖点、仓位建议 → 拒绝投资建议,可改为市场级信息梳理。
- 无法判断时,先澄清用户想看「市场整体影响」还是「具体公司/行业/板块深度分析」。

完整分类、默认时间窗、模糊场景和转接话术见 `references/question-types.md`。

## 默认交付契约

命中本 Skill 后,**默认不要生成 HTML、artifact、网页文件或可视化页面**。默认交付物是直接出现在对话框里的市场解读正文,正文必须能独立阅读,不能只说「已生成简报」或只给文件入口。

**默认顺序**:

1. 判定问题类型、市场范围、时间窗和是否越界。若越界或模糊,先转接或澄清。
2. 检索取证。优先官方、一手或权威财经来源;保留标题、来源、发布时间、链接和关键事实。
3. 去重合并。合并同事件多源,剔除弱相关、过时和传闻;传闻默认不进入前台。
4. 直接写对话框正文。不要为了本地普通问答强制构建完整 `briefing.json`,也不要默认读取 HTML 模板、schema 或构建脚本。
5. 正文结尾提示:必须逐字输出「需要为你生成一份精美的热点报告么？」,不得改写、润色或补写「HTML 格式」「解读简报」等词。只有用户明确要求 HTML/网页/可点击简报/导出文件,或回答需要,才进入 HTML 路径。

**外部模型例外**:豆包、Coze、Canvas、浏览器插件等外部承载场景不得让模型自由写 Markdown 报告。外部模型一律按 `references/doubao-json-writer.md` 填完整 `briefing.json`;新建 JSON 时先用 `scripts/new_briefing.py` 生成脚手架;用户要求 HTML 或宿主要求文件交付时,使用 `scripts/run_external_delivery.py` 校验并生成 HTML 和对话框正文。在交付 HTML 产物时，务必遵循下面的顺序：先使用 FileBatchUpload 工具拿到公网链接，作为附件；将附件带给 app_builder_agent 工具进行交付，要求严格按照 HTML 源文件进行渲染输出，不做任何代码变更，技术栈严格使用html，即arch_type=html。这样默认对话正文和 HTML 展开版共享同一份事实、同一组引用和同一条 7 段叙事链。

**对话框正文结构必须按 `references/dialog-delivery.md` 执行,并同构于 HTML 的 7 段叙事链**:

- 核心观点:用「核心判断」一句话直接回答用户问题,再用 30 秒结论解释事实变化、市场含义、限制条件和主要分歧。
- 有哪些关键增量信息？:只写新增事实、日期、对象、数字或市场反应,不要写过程提示。
- 这些信息为什么重要？:讲清真正变化、预期差和市场正在重新定价的变量。
- 上述信息如何影响市场？:按「事实 → 变量 → 资产/市场反应 → 后续验证」拆解传导。
- 各方有哪些值得关注的观点？:保留官方事实、市场定价、机构观点和反向条件,不要只给单边结论。
- 哪些信号会验证核心观点？哪些信号代表观点证伪？:分别写确认主线和削弱/推翻主线的信号。
- 信息来源:列出支撑关键依据的核心来源,包含来源名、日期和原文链接。
- 边界:说明这是公开信息梳理和验证条件拆解,不构成投资建议。
- HTML 提示:默认结尾必须逐字输出「需要为你生成一份精美的热点报告么？」,不得改写、润色或自行开始生成 HTML。

不得把正文改成自拟报告目录,例如「一、政策背景」「二、当前立场」「三、市场分歧」「四、监测表」「五、核心结论」。这些目录会改变 HTML 已定义的信息架构,导致对话正文和展开页叙事不一致。

市场级的是非 / 预测 / 「会不会、是不是、要不要、分析一下」类问题也按上述正文结构回答。不要只给一句预测;要把判断写成条件推演,说明哪些事实支持、哪些信号会推翻。

## 控制上下文占用

长时间运行时,优先保护最终对话框正文的空间。执行以下规则:

- 不要一次性读取所有 references、模板、schema 和脚本;按需读取。
- 默认正文路径不读取 `templates/`、`references/briefing-schema.json`、`references/briefing.schema.json`、`references/style-guide.md` 或 HTML 构建脚本。
- 检索笔记只保留可支撑正文的事实原子;不要把长网页摘录、执行记录、搜索过程或内部推理带到前台。
- 来源通常控制在 3-6 个核心公开来源;复杂宏观题可更多,但正文只展示最关键来源。
- 如果检索或校验耗时很长,收敛到「核心判断 + 依据 + 后续验证 + 边界」,不要继续扩展页面构建链挤占回复空间。
- 不输出 CoT、内部推理、执行记录、文件编辑记录或「修复第 X 行」之类过程。

## 资源加载顺序

按任务需要渐进读取,不要一次性塞满上下文:

- 归类问题与默认时间窗:读 `references/question-types.md`。
- 判定分析模式:需要时读 `references/analysis-modes.md`,设置或推断 `analysis_mode`。
- 检索前:读 `references/search-playbook.md` 和 `references/source-verification.md`。
- 写洞见前:读 `references/insight-framework.md`;控制篇幅时读 `references/reading-budget.md`。
- 默认对话框输出前:完整读取 `references/dialog-delivery.md`;它是对话框输出的唯一 Markdown 契约,顶层栏目必须和 HTML 同名同序。
- 输出和合规前:读 `references/compliance.md`、`references/chinese-copy-style.md`。
- 用户要求或确认生成 HTML 时:读 `references/html-delivery.md`,再按需读取 schema、模板和脚本。
- 豆包、Coze、Canvas、浏览器插件等外部 JSON 填写场景:读 `references/doubao-json-writer.md`。

## 检索与分析流程

1. **理解问题**:判定市场范围、时间窗、交易时段和分析模式。缺时间窗时,全市场默认近 24h;宏观/政策/风险事件默认从事件发布起或近 7 天,取更能覆盖关键事实的窗口。
2. **信息检索**:见 `references/search-playbook.md`。围绕「事实变化 / 来源可信度 / 市场含义 / 反向证据 / 后续验证」五问检索。现代人物、政策、价格、市场数据、新闻和日期必须用最新公开信息核验。
3. **去重降噪**:同事件多源合并为一条,保留最权威来源和印证来源;同一网页只作为一个来源。二手转述与传闻默认丢弃;确需纳入时只以争议点或官方回应呈现。
4. **重要性排序**:不打数值分。按相关性、时效、来源可信度、市场影响和新颖度排序;可内部标高/中/低和上榜理由。
5. **洞见生成**:回答核心观点、3-4 个关键证据、为什么现在重要、真正变化、各方分歧、验证/证伪信号。洞见是市场解读,不是买卖建议。
6. **对话框交付**:按 `references/dialog-delivery.md` 输出深度 Markdown 正文,引用关键来源和原文链接,最后一行逐字输出「需要为你生成一份精美的热点报告么？」。
7. **可选 HTML**:只有在用户明确要求或确认后,按照 `references/html-delivery.md` 生成 `briefing.json` 和 HTML。需要生成 HTML 的场景,严格按照 skill 里的格式要求生成成品 HTML;交付 HTML 产物时,先使用 `FileBatchUpload` 工具拿到公网链接作为附件,再将附件带给 `app_builder_agent` 工具进行交付,要求**严格按照 HTML 源文件**进行渲染输出,不做任何代码变更。

## 兜底

- 检索不足 → 明确说「暂无足够公开信息支撑」,给出已经核验到的事实、缺口和可继续搜索的检索词,不要硬编。
- 无 baseline 的「更新追踪」 → 降级为近 7 天默认,并提示基准时间。
- 问得过宽但仍是市场级 → 默认给全市场简洁解读;问得过深且落到具体公司/行业/板块 → 先澄清或转相邻 Skill。

## 自检

交付前确认:问题没有越界;关键事实有来源和时间;来源尽量一手/权威;传闻没有当前台事实;正文直接回答用户问题;顶层栏目与 HTML 同名同序,至少包含核心观点、有哪些关键增量信息、这些信息为什么重要、上述信息如何影响市场、各方有哪些值得关注的观点、验证/证伪信号、信息来源和边界;没有 Markdown 表格、过程提示、荐股、目标价、仓位或涨跌承诺;没有执行记录和内部推理;默认没有生成 HTML;结尾已逐字输出固定提示「需要为你生成一份精美的热点报告么？」。
