# 中文财经简报文案规范

目标:输出像中文财经媒体的简报,不是英文 dashboard 的直译。

## 标点与空格
- 使用中文全角标点:逗号`，`、句号`。`、顿号`、`、冒号`：`、分号`；`、括号`（）`、引号`「」`。
- 中文与数字、英文缩写之间不加空格:近7天、24小时、A股、CPO板块、3条重点。
- 百分号、倍数、时间不加空格:15%、2倍、6月11日15:30。
- 不使用英文箭头和装饰符号作正文连接。必要时用「影响链」「后续看点」等中文栏目承载。

## 日期与时间
- 前台日期优先显示为`6月11日`或`6月11日15:30`。
- 时间范围显示为`6月4日—6月11日`。
- 保留 `briefing.json` 中的 ISO 或 `YYYY-MM-DD HH:mm` 给渲染端解析;前台由模板转换。

## 栏目与按钮
- 「Headline」写作「一句话主线」。
- 「Highlights」写作「核心要点」。
- `news_items[].why` 作为事实说明素材使用,前台不写「上榜原因」或「入选原因」标签。
- 「Original」写作「查看原文」,且必须渲染为带 `href` 的 `<a>` 链接,不能只是普通文字。
- 「Official / Media / Institution」写作「官方」「媒体」「机构」。
- 不在前台出现「unverified」「待核验」「source」「corroboration」等内部词;分别转写为「争议点」「来源」「多源印证」。

## 句式
- `news_items[].why` 写标题之外的依据、影响范围和限制条件,不要复述标题。
- 少用抽象判断,多写「发生了什么」「影响什么」「后续验证什么」。
- 不写目标价、仓位、买卖点、涨跌承诺。
- 来源、日期和「查看原文」由模板展示,正文不要写「某某来源公开信息显示」「据报道」「发布时间为」。
- 影响链里把事实、传导逻辑和市场含义写成对应节点标题下的一段正文。

## 通俗易懂硬规则

最终 HTML 面向普通股民,不是写给研究员或设计评审。所有前台文案都要满足:

- 先说结论,再说原因。
- 一句话只讲一个意思,尽量少用长定语。
- 能说「市场担心钱会流走」,不要说「流动性约束边际强化」。
- 能说「后面主要看成交额能不能继续放大」,不要说「关注量能持续性验证」。
- 避免字段名式表达,如「正向条件」「负向条件」「风险因子」;改成「确认主线」「推翻或削弱主线」「接下来盯什么」。
- 专业词第一次出现要顺手解释,例如「折现率上升,简单说就是未来利润被打折得更厉害」。
- 来源说明写清「这条来源支撑了什么」,不要只堆媒体名。
