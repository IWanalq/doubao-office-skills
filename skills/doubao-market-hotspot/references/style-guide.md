# 视觉与一致性规范

## 颜色语义(中国习惯,务必区别西方)
- **红 = 涨 / 利好**(`#E4393C` 系),**绿 = 跌 / 利空**(`#2BA471` 系),中性灰 `#6B7280`。
- 来源级别徽标保持中性,A/B/C 不用强色抢主线;来源可信度主要靠来源名、域名、印证数表达。
- 结构主色使用墨蓝灰(`#293241` 系),只给链接、引用和结构性强调;页面大背景保持纯白(`#FFFFFF`),内容卡用白底/近白底区分层级,不要用整页蓝底或绿底。
- 主强调色使用温暖琥珀/锈色(`#A15C16` 系),用于新信息徽标、阅读进度和少数强提示。
- 重要性档:高=墨蓝灰浅底、中=琥珀、低=灰;不要用红色表示高重要性,避免和涨跌语义冲突。
- 内容主题色允许丰富但必须有语义:CPO/算力=青蓝,人形机器人=紫,黄金/宏观=金,市场反应=玫红,政策/官方=绿。不要整页只靠品牌蓝。
- 大面积背景必须克制:页面底色默认纯白,卡片通过细边框、近白底、轻阴影建立层级;主线 hero 只允许 3-4px 细语义色条,不能使用抢眼彩虹边框或大面积高饱和渐变。
- 颜色职责要清楚:结构主色只给链接、引用和结构性强调;强调色只给少数提示;主题色只给新闻/要点/摘要卡的标签和浅底;涨跌红绿只给金融方向,不要拿来装饰普通组件。

## 顶层按钮与块
- 顶部不渲染导航栏、复制摘要或打印按钮;首屏直接从报头进入核心观点,避免把阅读路径切碎。
- block 以「白底 + 细边框 + 轻阴影 + 语义左边线」建立层级。少用厚重描边、重投影和同色系填满背景。
- 核心要点、摘要卡、新闻卡根据标题/内容自动套用语义主题色;同一屏最多同时出现 3 个强主题色,其余用中性色承接。

## 排版
- 字体:`-apple-system, "PingFang SC", "Microsoft YaHei", sans-serif`;数字等宽便于对齐。
- **适老化基线**(散户老年用户多):正文最小 15px,关键数字 ≥20px,正文/背景对比度 ≥ 4.5:1。
- 信息密度:第一屏只放核心观点和 3-4 个关键证据;关键证据不显示 1/2/3 编号,不显示涨跌箭头,只用关键数字/日期/重点词做扫读锚点。关键增量信息默认露出 3 条,其余折叠。移动端单列滚动,不用横向 Tab。
- 重要性模块只回答「相比上一阶段或市场预期,真正变化在哪里」,不要复述首屏核心观点,也不要重复新闻列表。文字要像人话,不堆术语(见 `insight-framework.md`「写成人话」)。

## 组件
核心观点 hero(标题不带句末标点)、关键证据状态卡(不显示编号和涨跌箭头,只强调关键数字/日期/重点词)、标题栏信源范围、关键增量信息列表、重要性三栏卡、影响链因果步骤卡(内容自适应高度·桌面优先每行2张·卡内编号+角色标签·同行右向箭头·前瞻虚线卡·每步标题下显示一段节点正文)、时间线(嵌入关键增量信息区)、各方观点列表(角色+观点+出处)、验证/证伪信号卡、紧凑信息来源账本(默认 5 条核心来源,其余折叠,不显示数量摘要句)、固定免责页脚(整体文本块,不编号)。

## 关键数字
- 标题、导语、关键增量说明、洞见、影响链里的关键数字(金额/百分比/倍数/`1.6T`/年份等)自动加粗等宽强调(`.fig`,同一中文字体+tabular-nums),让数字成为扫读锚点;金额建议带千分位。不要为了数字强调切换宋体/衬线字体,避免和正文中文字体不一致。
- 关键证据卡不显示方向箭头。方向含义留在正文解释里,视觉上只强调关键数字、日期或 `【重点词】`,避免标签、箭头和值挤在一起。

## 信任结构
- 顶部只保留标题栏的 `meta.source_scope` 信源范围说明;首屏核心观点下不再渲染「主要来源」脚注行。详细来源、域名与原文链接由正文引用 `[n]` 的 popover、新闻卡、观点卡和底部「信息来源」账本承载。
- `interpretation[]` 渲染为「各方有哪些值得关注的观点？」,把各方观点(客观事实/市场反应/机构观点/反向证据,必要时补充媒体报道)摆在一起、可保留分歧;每条只显示 `point` 正文并在句末展示引用,不再生成截断式小标题,避免看起来像 AI 自说自话。
- 新闻卡必须展示 `category` 分类标签,帮助扫读;分类标签保持中性灰,不和行情红绿混用。

## 关键增量列表
- 「有哪些关键增量信息？」固定渲染为纯文本事实列表,不在条目前置图片或预留图片位。
- 不给 `news_items` 填 `image`;机器 schema 不接受该字段,构建会拒绝这类输入,避免破图、空白图框、外链失败或泛化配图抢占核心信息。
- 关键增量信息区优先完整展示 `highlight.lead_ref` 对应新闻;标题、两句事实说明、分类、来源、时间和原文链接必须独立完整可读。
- 不为了版面填泛化配图。K 线背景、芯片背景、抽象金融图、城市夜景等一律不用。

## 交互
- 长简报保留阅读进度和回到顶部按钮,不再使用粘性栏目导航。
- 重点信息为静态列表:默认露出 3 条,其余通过「展开」查看;不提供搜索、排序、筛选、密度切换或分享卡等控件。
- 影响链按 `visuals.impact_chain` 顺序渲染为内容自适应的因果步骤卡(每行最多 3 张、可排两行;卡内含编号 + 起因/传导/结果/前瞻角色标签,同行卡片间用右向箭头连接,末项做成前瞻虚线卡)。`visuals.impact_facts` 必须尽量带 `step`,HTML 会把 `fact` 作为对应步骤标题下的一段节点正文显示;正文要自然讲清这一步为什么成立、对市场定价意味着什么,并在句末带引用,不要拆成「变量变化 / 市场含义 / 支撑事实」字段。不要强制等高,避免短文本卡片出现大块空白。
- 前台不展示「待核验」。D 类/传闻类信息默认不进入新闻卡,确需纳入时写成「争议点」或「官方回应」。
- 页面必须按固定 7 段叙事链展示,不要把重要性、观点或验证信号提前混到首屏。
- 引用 `[n]` 默认用原地 popover 展示来源摘要、域名与原文链接,不要让用户跳到折叠区后丢失阅读位置。
- 仅在 `time/date` 可解析时显示相对时间与「新」标;不可解析时间原样展示,不硬算。
- 展开其余重点信息属于默认操作;离线打开时也必须可用。
- 所有可点击控件用 `button` 或可聚焦元素,提供 `:focus-visible`;动态计数用 `aria-live`;动画遵守 `prefers-reduced-motion`。

## 一致性四保障
1. 单一 `briefing.json` 数据源(深度对话框回复 + HTML 同源)。
2. 统一 design token(颜色/圆角/间距写死在模板)。
3. `render.py` 注入,模型**不手写 HTML**。
4. `examples/` 留基准样例做回归。

## 空状态
某栏目无信息时显示占位文案(如「该方向暂无可靠公开信息」),不留空白、不编造。

## 图表
骨架版用内联 SVG + CSS 实现视觉摘要、影响链、时间线,**零外部依赖**,保证 render 后离线可开。不要依赖外链图片才能读懂简报。

## 可借鉴的仪表盘视觉

允许借鉴:
- 每段开头的「本段结论」提示条,但文案必须面向用户,不能写设计说明。
- 首屏关键证据指标卡:大数字/状态 + 短标签 + 来源引用。
- 验证/证伪信号卡:信号名 + 两句完整叙述,不使用「看什么 / 说明什么」字段标签。
- 来源区的轻量摘要行,只保留来源条数和来源类型,不要做成占屏的统计卡。
- 数据型问题可选微型条形图,表达成交额、资金流、概率变化等。

默认不采用:
- 大半圆仪表盘、火焰评分、emoji 评分。
- 默认 treemap/heatmap。
- 复杂筛选、排序、搜索、分享交互。
- 依赖 hover 才能读懂的信息。
