# 检索剧本(search-playbook)

整个 Skill 的价值取决于这一步的质量:**搜得准、去得净、排得稳**。本文件给可执行规则,不停留在原则。

## 0. 工具无关原则
凡需联网,统一用「宿主提供的搜索/抓取能力」。**不要**写死某个工具名,**不要**用 curl/python 绕过宿主去抓网页。搜不到 → 走兜底,不编造。

## 1. 定向信源(优先级,而非泛搜)
通用搜索对 A 股财经源覆盖差、噪音大。优先用「检索词 + 站点限定」检索:

| 级别 | 信源(站点) | 用途 |
|---|---|---|
| A 官方 | 巨潮 cninfo.com.cn、上交所 sse.com.cn、深交所 szse.cn、证监会 csrc.gov.cn、央行 pbc.gov.cn、公司公告 | 公告、政策原文、确定性事实 |
| B 主流财经媒体 | 财联社 cls.cn、证券时报 stcn.com、上海证券报 cnstock.com、中国证券报 cs.com.cn、第一财经 yicai.com、界面 jiemian.com、华尔街见闻 wallstreetcn.com | 新闻、快讯、解读 |
| C 行情/数据/研报 | 东方财富 eastmoney.com、同花顺 10jqka.com.cn、雪球 xueqiu.com(行情/研报摘要) | 数据、研报摘要(标口径与日期) |
| D 社区/情绪 | 东财股吧、雪球讨论、微博财经 | 仅作「情绪面」,**强制标待证实** |

检索时优先 A/B;C 用于补数据;D 仅作情绪参考。海外事件(英伟达等)取官方与一线外媒,再映射到全球资产、港股/A股市场情绪或 A 股**环节/方向**(不点名个股,见 compliance)。如果用户实际要单家公司或具体板块深度分析,转相邻 Skill。

## 2. 检索角度(按问题类型展开,每角度 2-3 个中文 query 变体)

本 Skill 只做市场级检索。不要把问题展开成单家公司基本面、板块热度量化或行业长期研究。

- 全市场热点 / 交易主线:指数与风格、量能、涨跌家数、行业扩散、领涨领跌方向、重要消息、资金与情绪。
- 市场涨跌归因:当日指数表现、成交变化、风格切换、权重资产、涨跌停结构、政策/外围市场/资金面触发因素。
- 政策/宏观:政策原文、权威解读、影响变量(利率、汇率、信用、通胀、盈利预期)、受影响方向/环节、历史可比、市场预期。
- 新闻/风险事件:事件事实、直接影响资产、避险链条、反向证据、官方回应、后续验证节点。
- 跨资产联动:起点资产(美股、港股、美元、美债、原油、黄金等)、传导变量、受影响市场、相关性是否稳定、反向信号。
- 风险偏好/市场情绪:成交、波动、涨跌停、行业扩散、ETF/北向/融资等资金线索、避险资产、情绪指标。

变体示例:「今天 A股 下跌 原因 成交量」「央行 降息 权益市场 影响 site:pbc.gov.cn」「原油 大涨 通胀预期 全球股市」「地缘风险 黄金 原油 避险情绪」。

## 2.1 强制五问(防止新闻罗列)
每份简报必须覆盖五个问题。搜不到可靠信息时写入 `coverage_gaps`,不要硬编。

| 问题 | 目的 | 典型落点 |
|---|---|---|
| 事实变化 | 本窗口真正新增了什么,是否过时信息重炒 | `news_items` / `evidence_atoms` |
| 来源可信度 | 官方、媒体、机构、社区分别怎么说 | `references` / `source_check` |
| 市场含义 | 影响方向、环节、情绪还是预期 | `insight` / `interpretation` |
| 反向证据 | 有无降温、证伪、冲突或官方澄清 | `negative_case` / `conflicts` |
| 后续验证 | 接下来观察什么公告、订单、会议、数据 | `verification.confirm` / `verification.invalidate` |

## 2.1.1 分析模式检索补充

| analysis_mode | 额外检索角度 |
|---|---|
| market_event | 事件基线、市场已反应什么、直接受影响变量、反向证据 |
| policy_macro | 政策原文、权威解读、受影响变量、行业/环节映射、历史可比 |
| dated_catalyst | 确定日期、软窗口、数据发布时间、会议议程、前后验证信号 |
| earnings_event | 仅市场级财报季/业绩风险:市场预期、风格影响、指数权重、风险偏好;单家公司财报数字转「财报与业绩分析」 |
| theme_watch | 仅市场级交易主线:主题催化、资金是否扩散、持续性证据、降温证据、拥挤叙事;板块热度转「板块热度分析」,行业长期转「行业分析」 |

## 2.1.2 政策/宏观央行题强制核验

央行利率、通胀、就业、汇率和跨市场问题不能只靠新闻聚合页、预测平台截图或中文媒体转述。进入 `policy_macro` 时,先完成一手事实核验,再写市场解读:

- 核心来源优先级:ECB/Fed/CME 等一手或准一手来源优先;ECB 决议看 `ecb.europa.eu`,Fed 决议、会议纪要、主席/理事任职和会议日程看 `federalreserve.gov`,市场隐含概率可用 CME FedWatch、交易所或清算所页面。
- 当前任职人、会议日期、利率区间、利率上限/下限、点阵图/纪要发布时间必须用官方来源核对;不要沿用模型记忆里的过期主席、过期会议日历或过期利率区间。
- 概率必须带来源和时间戳:任何「维持不变概率 98.5%」「年内加息概率 52%」这类动态市场概率,都必须在 `evidence_atoms[].context` 或 `references[].date` 中写清 `as_of` 时间,并在 `verification` 或 `interpretation` 里说明它是市场定价而不是央行承诺。
- 若只能找到媒体转述,该事实不得承担核心结论;写入 `coverage_gaps`,并把结论降级为「暂无足够一手信息支撑」。
- 对「会不会跟随」类问题,结论要拆成三层:是否机械跟随、下一次会议的基准情景、什么美国本土数据或市场定价会改变判断。不要把外央行动作单独写成 Fed 决策主因。

## 2.2 内部审计字段
检索过程中维护以下内部字段;它们不进前台 HTML,但用于自检和复盘:
- `query_log[]`: `{query, intent, source_scope, result_count?, used_refs?}`。
- `evidence_atoms[]`: `{claim, source, url, date, excerpt, context, confidence, refs?}`。一条一事实;进入前台的关键判断应能回指到证据原子。
- `coverage_gaps[]`: `{angle, reason, next_query?}`。某角度无可靠信息时记录缺口。
- `conflicts[]`: `{claim, source_a, source_b, nature, handling}`。不同口径并存时保留冲突,不要压平。

## 2.3 图片处理(关键增量不配图)
「有哪些关键增量信息？」固定是纯文本事实列表,不要把图片放进 `news_items`:
- 禁止:给 `news_items[].image` 填官方图、原文配图、截图、AI 图、泛化股市 K 线、芯片背景、城市夜景、抽象金融图或任何 stock 图。
- 原因:关键增量区承担事实扫读,外链图片容易破图、加载慢、占用首屏,还可能被误读成证据本身。
- 契约:`briefing.schema.json` 不接受 `news_items[].image`;如果输入里出现该字段,构建会拒绝。
- 可记录:真正有信息量的图表,用文字总结进 `why`、`visuals.impact_facts` 或 `references[].title/source/date/url`,不要在前台新闻卡前置图片。

## 3. 发布时间处理(关键)
- 每条尽量取到**可信发布时间**;写入条目。
- 取不到可信日期 → **降权**并标 `时间存疑`,不得当作新信息置顶。
- 超出时间窗的过时信息剔除;事件追踪保留时间线节点。

## 4. 去重与合并
- 同一事件多源 → 合并为**一条**,保留最权威来源作主引用,其余计入 `corroboration`(印证源数)。
- 同一网页多事实 → `references[]` 只保留**一个** id;不同事实、观点、影响链和验证信号都回指这个 id。不要把同一篇文章按段落或用途拆成多个来源 item。
- URL 去重按 canonical URL:去掉 `utm_*`、`spm`、`from`、fragment、大小写、`www.`、默认端口和末尾斜杠;保留会改变正文内容的查询参数。
- 转载、洗稿、标题党 → 去重剔除。
- 二手转述/传闻 → 默认丢弃;传闻是否纳入见 compliance(仅官方回应或重大多源时纳入)。确需纳入时写成「争议点 / 官方回应」,前台不展示「待核验」。

## 5. 排序(离散档,不打数值分) {#排序}
对每条按 5 维**定档**,不做加权算术(数值分跨模型不稳)。

| 维度 | 高 | 低 |
|---|---|---|
| 相关性 | 与主题直接相关 | 擦边/泛行业 |
| 时效性 | 24h 内/突发 | 超窗/过时信息 |
| 来源可信度 | 官方/交易所 | 社区小作文 |
| 市场影响 | 影响整条产业链/指数 | 个别噪音 |
| 新颖度 | 首次披露/打破预期 | 反复报道(无 baseline 时降权) |

**定档规则**:命中「相关性=高 且 (时效=高 或 来源=高)」→ `importance:高`;两项中 → `中`;否则 `低`。每条必须写 `why` = 命中的 1-2 个维度的自然语言说明。**排序优先级**:相关性 > 时效 > 来源 > 市场影响 > 新颖度。默认取 Top 5-8。

## 6. 检索预算(保证「每天刷」的速度)
单次简报建议上限:搜索查询约 8-12 次以内,页面抓取约 6-8 篇以内。先广度覆盖各角度各 1 次,再对将进入 L0 的 2-3 条做深度抓取。达上限即停,用已得信息成稿,缺口在简报里显式说明。

## 7. 兜底
- 某角度无命中 → 简报标「该方向暂无可靠公开信息」。
- 整体信息不足 → 输出「暂无足够公开信息支撑」+ 建议检索词,不硬编。
