# 阅读预算

目标:让用户在 1-3 分钟内读到有效信息。HTML 页面多余信息宁可折叠,不要铺满页面;默认对话框正文没有折叠能力,必须把 HTML 的核心叙事浓缩成可直接阅读的 Markdown。

## 三层阅读
- **30 秒**:核心观点 + 3-4 个关键证据。用户能回答「结论是什么」和「为什么敢这么说」。`reading_budget.thirty_second` 必须承载事实依据、市场含义、限制条件或后续验证,不能只写一句结论。
- **1 分钟**:关键增量信息 + 为什么重要 + 影响链。用户能回答「新增事实是什么」「真正变化在哪里」「如何传导到市场」。
- **3 分钟**:各方观点 + 验证/证伪信号 + 信息来源。用户能回答「哪些观点有分歧」「后续验证什么」「来源支撑在哪里」。

## 写作要求
- `news_items[].why`:保持为可核对的事实说明,补充标题没说清的依据、影响范围和限制条件,不要复述标题。
- `reading_budget.thirty_second`:写成一段核心观点,同时回答结论、关键依据、主要分歧或后续验证。
- `section_summaries.impact / viewpoints / verification`:字段名保留,但语义是标题下方的关键洞见。不要写栏目摘要或泛泛提示「继续跟踪」;要写判断、条件和证据边界。
- `visuals.impact_facts[].fact`:写成对应影响链步骤标题下的一段节点正文,自然讲清这一步为什么成立以及还需要什么事实验证。
- `interpretation[].point`:每条必须有 `refs`,并写清该角色提供了什么证据、判断和反向条件。

## 校验等级
- `scripts/briefing_validation.py` 不对字段文案做语义打分,也不会因为文案表述方式而阻断交付。
- 校验失败只表示结构、引用、来源可信度、安全边界或 HTML 交付契约存在问题。每条阻断问题只给 `path/message`。
- 收到 `advisory_issues` 时不要重写或删减整份 JSON。advisory 只提示版面或填写建议,不触发修复循环。
- 修复时只修结构、引用、来源或交付契约错误,不要把纯文案润色当成校验修复。

## 版面规则
- 首屏只保留主线和核心要点;信源范围只在标题栏 `source_scope` 展示,不要铺满长新闻列表。
- 不再渲染「主要来源」脚注行;详细来源由正文引用 popover、新闻卡、观点卡和底部「信息来源」账本承载。
- 默认展示 3 条重点信息,其余放「展开」;重点信息为静态列表,不带筛选/排序/搜索/分享控件。
- 信息来源默认展示最多 5 条核心来源,其余折叠;核心来源优先放一手/官方/权威来源,不在页面上额外写“默认展示X条”这类摘要句。
- 顶部不放导航、复制摘要或打印按钮;阅读路径从核心观点自然下滑到后续栏目。
