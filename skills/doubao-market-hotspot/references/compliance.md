# 合规与脱敏(不可删减)

面向中国股民,非持牌不得提供具体投资建议。本 Skill 只做「公开信息聚合 + 中性解读」。

## 红线(硬规则)
- **不**荐股、不给目标价、不给买卖点、不给仓位建议、不承诺涨跌。
- 个股只能作为**新闻主体**或**产业链相关项**出现,不附带任何买卖含义。
- 不处理、不传播非公开调研、内幕、操纵相关内容。

## 脱敏规则
- **政策 / 事件 → 只到「可能相关方向 / 环节」**,不输出受益股名单。
  - 反例(禁止):「利好 XX 股份、YY 科技」。
  - 正例:「利好光模块、铜连接环节,相关产业链需关注放量节奏」。
- **海外映射(如英伟达→A 股)**:同样只到环节/方向,不点名个股。
- **公司相关事实**:只有当公司新闻已经成为市场级事件的支撑事实时才可出现,且只陈述事实和市场传导;要个股涨跌或基本面 → 转「个股日报」或「公司分析」。

## 传闻纳入规则
传闻/未证实信息**默认丢弃**;仅在满足以下任一时纳入,且必须写清官方回应状态:
1. 已有官方回应(澄清/确认);或
2. 重大且被多个独立来源报道。

前台不得展示「待核验」标签。确需提及时,改写为「争议点」「官方回应」或「尚待公告确认」,并降低展示层级。

## 敏感话题
政策 / 宏观表述保持中性,**只引官方口径**,不做政治性引申或评判。

## 固定尾部(每份简报必带)
- `disclaimer_id` 必须使用 `standard_v2_full_risk_notice`。该版本的正文以 `references/disclaimers.json` 为准,由 `render.py` 注入 HTML,模型不得手写、删减、改写或用短免责声明替代。
- HTML 尾部必须以独立区块展示以下完整文本,渲染为一个整体文本块,不得拆成三条色块,不得加 1/2/3 编号,不得在三段前加 `-`、`*`、`•` 等项目符号:
```
风险提示与免责声明：
以上内容为AI自动生成或AI辅助生成，仅用于信息整理、投研辅助、教育交流或一般性分析参考，不构成对任何金融产品、交易策略或投资行为的推荐、邀约、承诺或保证，也不构成投资、法律、税务、会计等专业意见。
以上内容可能基于公开信息、历史数据或用户提供材料进行总结、归纳、推演与情景分析，但相关内容可能存在时效性不足、信息缺漏、事实误差、模型偏差或生成性错误，历史数据、历史业绩、回测结果及情景假设均不代表未来表现。
用户应基于自身风险承受能力、投资目标、财务状况及适用法律法规独立作出判断，必要时咨询持牌专业机构或顾问。任何因依赖以上内容而作出的决策及其后果，由用户自行承担。
```
- 时间窗、交易时段和信源范围只在页面顶部标题栏展示,不要在风险提示区块底部重复展示「覆盖范围」。
