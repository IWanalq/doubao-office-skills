## 数据源优先级与失败兜底

## 总原则

行情、新闻、预测、资金流都是高时效数据。执行本 skill 时必须联网或调用可用行情工具，不要凭记忆。

执行顺序硬规则：

1. 必须先获取、交叉验证并形成内部数据核验记录，再展开后续分析。行情数据包括：当前/收盘报价、成交额、换手率、量比、市值、PE、资金流、板块、MA、MACD、RSI、压力/支撑等；当前版本不再获取 30 个交易日日线 OHLCV。内部记录不要求保存为本地文件，且不得作为文末信息来源。
2. 当前版本不获取 1 分钟级行情，不获取 90 个交易日行情，不生成 K线图，不生成 `window.RECAP_CHART_DATA`。
3. 关键数据原则上用 2-3 个独立渠道核验。只有证券代码、交易市场、日期/时间、单位、复权口径和方向一致，才允许进入报告；冲突或可疑时宁可留空。
4. 如果行情数据不完整、不可信或字段映射不确定，不得进入正文分析和 HTML 生成；必须换数据源、扩大请求区间、修正市场代码或分页参数，循环尝试直到数据完整。至少 3 次尝试仍失败时，标记缺失并跳过对应展示/分析。
5. 所有进入报告的数值都必须能追溯到可打开来源链接。行情摘要、资金流、板块行情、技术指标和一致预期不得以“接口数据”名义免引用。
6. 在豆包等支持专用搜索工具的环境中，取数和资料搜集必须先调用 `seed_finance_search` 与 `general_search`。`seed_finance_search` 优先用于证券识别、行情、资金、板块、指标和预测等金融结构化数据；`general_search` 优先用于公告、新闻、产业信息、权威来源原文、数据页链接和交叉验证。
7. `seed_finance_search` / `general_search` 是优先入口，不是免检来源。两者返回的数据也必须与 MCP、券商、交易所、腾讯、Wind/Choice/FactSet/东方财富或其他可用来源交叉核对后才能验收。

## 行情数据源优先顺序

1. `seed_finance_search`：第一优先，用于证券识别、实时/收盘报价、成交额、换手率、量比、市值、PE、资金、板块、技术指标和一致预期等结构化金融字段；返回后仍需交叉验证并保留可打开来源链接。
2. `general_search`：第二优先，用于辅助定位证券代码、验证行情口径、寻找权威行情页、公告页、新闻原文和产业信息。
3. 同花顺 MCP / 同花顺行情接口：当 seed/general 不可用、字段不足或校验失败时，用于 A股、港股和可覆盖市场的实时/收盘报价、成交额、换手率、量比、市值、PE、资金和技术指标等字段。
4. 华泰证券 MCP / 华泰证券行情接口：作为同等级 MCP 备选，用于可覆盖市场的实时/收盘报价、交易结构、资金和技术指标数据。
5. 老虎证券 MCP / 老虎证券行情接口：优先用于美股、港股及其可覆盖市场的实时/收盘报价、成交量、估值和技术指标数据。
6. 腾讯自选股/腾讯行情接口：当上述工具和 MCP 不可用、返回字段不全或校验失败时使用；适合 A股、港股的实时报价、成交额和部分行情字段，通常比东方财富、Yahoo 更稳定。
7. AkShare / Tushare：环境可用时作为结构化备选，尤其适合 A股。
8. 东方财富：适合资金流、公告、盈利预测、板块和部分行情；但部分环境可能返回 502/567/空响应，不要只依赖东方财富。
9. Yahoo Finance / yfinance：适合美股和部分港股，但可能限流；失败时切换其他源。
10. Google Finance、Stooq、交易所、券商 API 或其他可用稳定源：作为最后行情兜底。

无论使用哪一个行情源，都必须先通过 `data-contract.md` 的字段、时间口径、单位和来源链接校验；MCP 返回的数据如果缺少关键摘要指标、技术指标或可打开来源链接，也必须继续切换下一数据源，不得因为来源优先级高就放宽门槛。

## 新闻、公告、预测等信息源优先顺序

1. `general_search`：优先用于检索交易所公告、公司公告、官方 IR、监管披露、权威财经媒体、产业媒体和新闻原文。
2. `seed_finance_search`：用于补充结构化金融信息、研报摘要、一致预期、板块和资金字段。
3. 交易所、公司公告、官方 IR、监管披露。
4. 权威财经媒体与产业媒体：财新、彭博、路透、证券时报、上海证券报、中国证券报、证券日报、财联社、第一财经、21世纪经济报道、Reuters、Bloomberg、CNBC，以及对应产业的专业媒体/研究机构。
5. 东方财富、Choice、Wind/彭博/FactSet/Refinitiv（如果环境可用）用于资金、预测、研报和板块。
6. 不采用：雪球、股吧、贴吧、微信群截图、自媒体小作文，除非只作为“市场传闻”并明确不可核验，通常不要写入报告事实。
7. 证券时报数据宝、东方财富 Choice 等“成交额/大宗交易/资金流”类短讯只可作为第一部分交易结构素材，不得充当第四部分消息面主来源。

## A股数据优先路径

### 1. 股票识别

优先：

- `seed_finance_search` 的证券识别、合约元数据或行情入口。
- `general_search` 辅助确认股票简称、代码、交易所和同名公司歧义。
- 同花顺 MCP / 华泰证券 MCP / 老虎证券 MCP 的证券搜索、行情元数据或合约查询能力。
- 腾讯行情代码：`sz300308`、`sh600519`、`hk00700`。
- 东方财富搜索/行情页。
- AkShare：如果 Python 环境装有 `akshare`，可用 `stock_info_a_code_name` 或相关接口匹配名称和代码。

### 2. 今日实时/收盘行情

优先：

- `seed_finance_search` 的实时/收盘行情或快照行情。
- `general_search` 定位权威行情页并交叉验证交易日期、市场和代码。
- 同花顺 MCP / 华泰证券 MCP / 老虎证券 MCP 的实时行情或快照行情。
- 腾讯 `qt.gtimg.cn` 或 `web.ifzq.gtimg.cn`。
- 东方财富 push2 quote。
- AkShare 实时行情接口。

需要字段：

- 最新价/收盘价、涨跌幅、开盘、最高、最低、成交量、成交额、总市值、PE TTM、换手率、量比。

如果接口只返回部分字段：

- 日内区间用最高/最低拼接。
- 成交额从接口直接取，无法取时用成交量和 VWAP 近似但必须在内部数据核验记录和口径说明中标注。
- PE TTM 缺失时用总市值/TTM归母净利润估算，缺 TTM 利润则表格或指标卡内写 `暂无`。

摘要指标核验：

- 当前价格/收盘价、涨跌幅、日内区间、总市值、市盈率 TTM、成交额、换手率、量比必须记录 `data_as_of`。
- 非交易日请求时，用最近有效交易日收盘数据；交易日盘中请求时，以请求时点之前可获得的最新快照数据为准。
- 每个字段至少尝试两个来源核对；若市值、PE、量比等字段来源分歧较大，优先保留可直接验证字段，分歧字段在表格或指标卡内写 `暂无`，并在内部数据核验记录中标注冲突，读者版不展开接口口径。

### 3. 技术指标、量比和关键价位

当前版本不拉取 30 个交易日日线，也不要求 agent 自行计算 MA/MACD/RSI。优先直接获取数据源披露的指标值：

- `seed_finance_search` 的 MA、MACD、RSI、量比、支撑/压力或技术指标摘要。
- `general_search` 辅助定位可打开的行情页、券商技术分析页、交易软件指标页、研报或公告口径。
- 同花顺 MCP / 华泰证券 MCP / 老虎证券 MCP 的技术指标、资金和交易结构字段。
- 东方财富、Choice、Wind、腾讯、Yahoo/Google Finance 等可打开行情页或数据页。

要求：

- 每个指标必须记录来源链接和时间口径。
- MA、MACD、RSI 至少用主来源披露值；尽量用第二来源核对方向。若不同来源算法差异较大，保留主来源值并在内部数据核验记录中说明差异。
- 压力位、支撑位和震荡区间优先采用可核验的技术分析页、研报、行情平台技术位或明确公式自行计算。自行计算必须引用输入数据来源，不得无来源生成价位。
- 若找不到可信来源，技术指标表格单元格写 `暂无`，正文可简短说明该指标暂未核验；不要为了版式完整而造数。

### 4. 资金流

A 股适用：

优先：

- `seed_finance_search`、同花顺/华泰/老虎 MCP 的资金或成交结构字段。
- 东方财富资金流向：主力净流入、超大单、大单、中单、小单。
- 沪深港通/南北向：交易所披露、港交所沪深港通数据、东方财富。

失败兜底：

- 主力资金无法取到时，资金流表格对应单元格写 `暂无`，或用可核验的大单成交占比/主动买卖盘替代。
- 不能随意编造净流入金额。

非 A 股处理：

- 港股、美股及其他非 A 股不生成 A 股资金流表，不硬套主力资金、北向资金、大单资金三行结构。
- 港股可用南向资金、成交额/成交量、换手、异常大宗交易等可核验交易线索。
- 美股可用成交量相对均量、盘前/盘后表现、期权成交/隐含波动率、ETF 流向、机构持仓变化等可核验交易线索。
- 非 A 股交易线索写入第一部分“价格与量能”文字分析；无可信来源时写“未见可核验的异常资金交易信号”。

### 5. 板块数据

优先：

- `seed_finance_search` 或同花顺/华泰/老虎 MCP 的行业/概念数据。
- 东方财富行业/概念板块行情。
- 同花顺行业/概念板块。
- 交易所行业分类。

输出：

- 主行业涨跌幅。
- 核心概念板块涨跌幅。
- 板块上涨/下跌家数。
- 个股相对板块强弱。

### 6. 技术指标

优先直接获取，不默认本地计算：

- MA5/10/20/60：采用来源披露值；若来源未披露 MA60，表格内写 `暂无`。
- MACD：采用来源披露的 DIF/DEA/MACD 柱体或等价口径；来源必须说明参数或使用平台默认参数。
- RSI：优先 RSI14；若来源使用其他参数，必须说明。
- 支撑/压力：采用可核验技术分析页、研报、行情平台技术位或带公式的自行计算。

## 消息和公告

优先：

- 巨潮资讯、上交所、深交所、港交所、SEC EDGAR、公司公告。
- 公司官网投资者关系。
- 财联社、证券时报、上海证券报、中国证券报、第一财经、财新、彭博、路透等权威媒体。
- 行业产业链用公司 IR、产业媒体和权威研究机构。

要求：

- 每条消息保留日期、来源、标题/摘要、原文链接、影响判断。
- 最近 3 日内的信息优先；太早的信息一般不进入每日复盘，除非用户明确追问或事件仍在发酵。
- 文末“信息来源”列出正文和表格实际引用的可打开原始外部来源，包括公告、新闻、产品、业绩、产业趋势、研报/一致预期，以及行情摘要、资金流、板块行情、技术指标、PE TTM、量比等数据来源。不得列内部核验记录、本地 txt、本地材料包或中间文件。

## 投行一致预期

优先：

- `seed_finance_search` 与 `general_search` 检索一致预期、研报摘要和可打开原始来源。
- Wind、Choice、FactSet、彭博、Refinitiv（如果环境可用）。
- 东方财富个股研报/盈利预测/机构预测页面。
- 券商研报原文（公司官网或研报平台）。

计算兜底：

- PE = 当前价格 / EPS。
- EPS = 归母净利润 / 总股本。
- 营收缺失不可臆造，表格内写 `暂无`。

说明：

- 不要把 EPS 估算出的净利润伪装成数据库直接披露值。
- A 股或披露完整盈利预测的数据源，表格可使用 `年份 / 归母净利润 / 营业总收入 / 每股收益 / 市盈率`。
- 港股、美股或主要披露券商评级/目标价的数据源，表格改用 `机构名称 / 最新评级 / 目标价 / 日期`，展示最近约 5 家机构，优先选择日期最新且来源可打开的记录。
- 投行一致预期表格下方必须写简短口径说明，例如“口径说明：A 股一致预期优先采用 seed/general 可核验来源及 Wind/Choice/FactSet/东方财富等数据库；缺失字段以暂无列示，自行计算项按 EPS、股本和当前价估算。”或“口径说明：港股/美股一致预期以近期可核验券商评级与目标价为主，展示最近约 5 家机构记录；目标价币种按原始来源列示。”
- 若没有可核验的机构覆盖、盈利预测、评级或目标价，不得输出全是 `暂无` / `未披露` 的投行一致预期表，也不得把公司公告、IR 或资讯聚合页伪装成投行覆盖。该节保留标题，正文说明暂无覆盖，并分析机会与风险：机会包括关注度低、预期发现、估值重估；风险包括外部跟踪不足、估值锚弱、信息透明度和流动性不确定。
- 不要把无法打开的接口地址作为“信息来源”来源行；必须尽量使用可打开的数据页、公告页、媒体原文、研报页或平台行情页。若只有接口且无法打开核验，该字段不得进入报告。

## 港股/美股差异

港股：

- 使用 HKEX、公司公告、东方财富港股、腾讯港股、新浪港股、同花顺/华泰/老虎可用接口。
- 北向资金不适用，可用南向资金。
- 币种通常为港币，市值和预测需要统一币种。

美股：

- 使用 Nasdaq/NYSE、SEC EDGAR、公司 IR、Yahoo Finance/yfinance、Alpha Vantage、Finnhub、Polygon（如可用）、老虎证券可用接口。
- 没有主力资金/北向资金口径时，不要硬套 A股资金流；可改为成交量、期权、机构持仓、ETF流向、盘前盘后表现。

## 本项目踩过的坑

- 数据抓错日期：先判断 `report_request_time` 和最近有效交易日，再取行情；周末、节假日、停牌时要向前推。
- 技术指标口径漂移：不同平台的 MA/MACD/RSI、量比、复权口径可能不同，因此必须记录来源、时间和平台口径；冲突无法解释时从缺。
- 数据无链接：只拿到数值但没有可打开来源链接时，不得进入报告；必须重新搜索数据页或权威来源。
- 来源角标遗漏：当前价、PE TTM、量比、资金流、MA/MACD/RSI、投行一致预期等数据都必须在正文/表格中有角标，并在“信息来源”中有对应“打开原文”。
- 新闻不要用小作文：没有权威来源时宁可写“暂未检索到权威消息”，不要引用股吧/雪球传闻。
- 投行一致预期可能缺字段：可以计算 EPS/PE，但必须标注估算口径。
