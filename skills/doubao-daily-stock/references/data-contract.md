## 数据字段与来源契约

## 核心原则

当前版本不生成 K 线图，不获取 1 分钟级行情，不获取 30 个交易日日线，不获取 90 个交易日行情，不生成 `window.RECAP_CHART_DATA`。所有数据只服务于对话版分析、HTML 摘要指标、资金表、技术指标、价格区间、评级和风险判断。

数据准确性高于完整度。任一字段若无法从可靠来源核验，宁可留空，也不得猜测、补造或沿用疑似错误数据。

硬规则：

- 每一个进入报告的数值字段都必须先进入内部数据核验记录，并记录原始外部来源。
- 不是 agent 自己计算的数值，必须有可打开来源链接；对话版和 HTML 中必须用上角标引用该来源。
- 如果是自行计算，必须记录公式、输入字段、输入来源链接和计算过程；报告中应标注为“自行计算”或在口径说明中写明。
- 行情摘要、资金流、板块数据、MA、MACD、RSI、PE TTM、量比、投行一致预期等数据都属于需要来源的数据，不得以“行情接口数据”名义免引用。
- 数据查不到来源时，必须回到取数步骤重新查；至少 3 次换源/修正口径后仍无可信来源，字段留空，不进入正文判断。

## 股票身份

输出字段：

- `company_name`：公司全称或证券简称。
- `ticker`：标准代码，如 `300308.SZ`、`0700.HK`、`NVDA`。
- `exchange`：交易所。
- `currency`：人民币/港币/美元等。
- `report_request_time`：用户发起请求时间，盘中精确到分钟；对话版和 HTML 开头必须统一使用该时间。
- `report_date`：报告生成日期，从 `report_request_time` 派生。
- `latest_valid_trading_day`：根据交易所日历和请求时间向前推算的最近有效交易日；非交易日、节假日、停牌时必须继续向前推。
- `data_as_of`：行情数据截至时间，盘中精确到分钟，收盘后写收盘日期和时间。
- `business_snapshot`：一句话业务定位，不超过 80 个中文字符。

股票身份也必须记录来源，例如交易所页面、公司公告、行情平台证券资料页或可信金融数据库。

## 内部数据核验记录

进入分析前必须先完成内部数据核验记录，作为后续分析和 HTML 的事实底稿。该记录不要求保存为本地 `.txt`，也不得作为文末“信息来源”。建议字段：

- 股票身份：简称、代码、交易所、币种、行业/板块、原始外部来源链接。
- 时间口径：`report_request_time`、`latest_valid_trading_day`、`data_as_of`、是否盘中/收盘后/非交易日。
- 行情摘要：当前价/收盘价、涨跌幅、日内区间、总市值、PE TTM、成交额、换手率、量比；每项记录数值、单位、时间、来源 1/2/3、链接、是否一致。
- 资金流：主力资金、北向/南向/外资或等价口径、大单资金或成交结构；每项记录来源与核验结果。
- 板块数据：最相关板块、当日涨跌、涨跌家数、龙头/权重地位；每项记录来源与核验结果。
- 技术指标：MA、MACD、RSI、震荡区间、压力位、支撑位；记录主来源、辅助核验来源、参数口径和是否允许展示具体读数。
- 新闻公告研报：来源、发布日期、标题、链接、是否在最近 3 日或报告要求区间内、与公司/行业关系。
- 一致预期：来源、字段、是否披露、是否自行计算、计算公式和原始输入来源。
- 缺失与剔除：所有冲突、可疑、无法打开、无法核验的数据和原因。

内部记录中未通过核验的数据不得进入报告正文、表格、评级理由或 HTML。

## 摘要指标卡

HTML 顶部的 8 个指标：

1. 当前价格/收盘价：`1180.00元`
2. 涨跌幅：`+2.17%`
3. 日内区间：`1126.90-1196.16元`
4. 总市值：`约1.32万亿人民币`
5. 市盈率 TTM：`88x`
6. 成交额：`385.2亿元`
7. 换手率：`2.91%`
8. 量比：`1.2`

要求：

- 只显示两行：标签 + 数值。不要第三行说明。
- 单位直接写进数值。
- 指标卡下方必须有 `.metric-note`，说明这些摘要指标的数据时间口径，例如：`注：以上行情摘要截至 2026-06-12 13:30（交易日盘中）。`
- 每个指标数值都必须带上角标，例如 `1180.00元<a class="cite-ref" href="#source-1">[1]</a>`。
- 若多个指标来自同一个可打开行情页，可以共用同一来源编号；若来自不同数据页，必须分别编号。

颜色规则：

- 本报告面向中国股民，红色代表上涨/净流入/正数，绿色代表下跌/净流出/负数。
- 涨跌幅为正数：`.positive`，颜色 `#9b2424`。
- 涨跌幅为负数：`.negative`，颜色 `#0f6b3f`。
- 涨跌幅为 0：默认色或 `.neutral`。
- 日内区间：`.neutral`，颜色 `var(--accent)` / `#143d59`。
- 其他指标使用默认黑色 `var(--ink)`。
- 资金流向表格中的净流入、正数金额、正数比例使用 `.positive`；净流出、负数金额、负数比例使用 `.negative`。

## 资金交易线索与板块

A 股资金流字段：

- 主力资金净流入/流出。
- 北向资金。
- 大单资金、大单成交占比或成交结构；无法获取时表格内写 `暂无`。
- 每项必须有金额/比例、时间口径、来源链接和复盘含义，不得凭空编造。
- 资金流表格中的每个数值必须带上角标；如果同一行多个数据来自同一来源，可在行内关键数值后引用一次。

非 A 股处理：

- 港股、美股及其他非 A 股不适用 A 股资金流表，不生成 `主力资金 / 北向资金 / 大单资金` 表格。
- 港股可记录南向资金、成交额/成交量、换手率、大宗交易、异常成交或机构持仓变化等可核验交易线索。
- 美股可记录成交量相对均量、盘前/盘后表现、期权成交/隐含波动率、ETF 流向、机构持仓变化等可核验交易线索。
- 非 A 股交易线索只进入第一部分“价格与量能”的文字分析，不单独生成资金流表格；无可信来源时写“未见可核验的异常资金交易信号”。

板块字段：

- 最相关的 1-3 个行业/概念/主题板块。
- 当日涨跌幅、涨跌家数或代表性龙头表现。
- 个股相对板块强弱：强于板块、弱于板块、跟随 beta 或独立 alpha。
- 板块名称、涨跌幅、涨跌家数和同业表现均需来源链接和上角标。

## 技术指标

技术指标优先直接获取权威数据源披露值，不再要求本地重算。

字段：

- MA：MA5、MA10、MA20、MA60。
- MACD：DIF、DEA、MACD 柱体或数据源提供的等价口径。
- RSI：默认 RSI14；若来源使用其他参数，必须写明。
- 震荡区间、压力位、支撑位：来自行情平台技术位、券商/研报、可核验技术分析页，或有公式说明的自行计算。

要求：

- MA、MACD、RSI 三行固定。
- 每行包括：指标、标签、分析、判断。
- `标签` 和 `判断` 为 2-6 个字短语；`分析` 为 1-2 句完整解释，照顾不懂技术指标的读者。
- 技术指标读数必须至少有主来源；尽量用第二来源核验方向。若不同平台读数不同，保留主来源读数并在内部数据核验记录中注明口径差异。
- 无法找到可信来源或读数冲突无法解释时，不展示具体读数；技术指标表格单元格写 `暂无`，正文可简短说明“该指标暂未核验”，并降低技术面结论确定性。

价格区间：

- 震荡区间：一个区间，如 `402-446元`。
- 压力位：可以是单点、多个单点并列，或一个区间；不得写两个区间并列。
- 支撑位：可以是单点、多个单点并列，或一个区间；不得写两个区间并列。
- 价格区间带图上只展示当前价、震荡区间低点、震荡区间高点；压力位和支撑位只写入下方三项列表。
- 当前价、震荡区间、压力位、支撑位均属于数据字段，必须有来源角标；若由 agent 自行计算，必须引用输入数据来源并标明计算口径。

## 新闻、公告与信息来源

正文重要事实和所有展示数据必须能回到真实来源：

- 最近 3 日内的公司公告、交易所/监管披露、公司 IR。
- 最近 3 日内的权威财经媒体、产业媒体、行业趋势新闻。
- 正文实际引用过的研报、一致预期或数据库页面。
- 当前价、涨跌幅、成交额、换手率、量比、资金流、板块、技术指标、PE TTM 等数据页面。

每条来源必须包含：

- 编号：`[1]`
- 媒体/来源名称
- 发表日期或数据日期
- 文章/公告/材料/数据页标题
- 行末右侧 `打开原文` 超链接

链接要求：

- `打开原文` 必须是真实可打开的外部原文、公告、监管披露、媒体报道、行情页、数据库页或研报页链接。
- 禁止空链接、`#`、`javascript:void(0)`、搜索结果页、虚构 URL。
- 正文每个上角标 `<a class="cite-ref" href="#source-N">[N]</a>` 必须对应文末 `id="source-N"` 的来源行。
- 不得把多个不同事实都挂到同一个来源编号，除非这些事实确实来自同一篇原文或同一个数据页。
- 凡数据查不到来源，必须回到取数步骤；仍查不到则从报告删除。若固定表格或指标卡必须保留该单元格，统一写短占位 `暂无`。

## 投行一致预期

优先数据源：

- `seed_finance_search` 与 `general_search` 检索一致预期、研报摘要和可打开原始来源。
- Wind、Choice、FactSet、彭博、Refinitiv、东方财富、券商研报或公司/交易所披露。

字段：

优先根据市场和数据可得性选择一种结构：

1. 财务预测表：年份、归母净利润、营业收入、EPS、PE。
2. 评级目标价表：机构名称、最新评级、目标价、日期。
3. 无覆盖说明：无表格，仅说明暂无可核验投行/券商覆盖，并分析机会与风险。

适用规则：

- A 股或数据源披露完整盈利预测时，优先财务预测表，通常覆盖未来 3 年。
- 港股、美股或数据源主要披露券商评级/目标价时，优先评级目标价表，展示最近约 5 家机构。
- 不得把两套表头混在一张表里。
- 若经过 `seed_finance_search`、`general_search` 及至少一个可用金融数据/券商/公告来源检索后，仍没有可核验的投行/券商盈利预测、评级或目标价，不得生成填满 `暂无` 的表格，也不得把公司 IR、公告或资讯聚合页当成机构覆盖。该节改为纯文字说明“暂无可核验覆盖”，并说明对买入该股票的机会和风险含义。

缺失处理：

- PE = 当前价格 / EPS。
- EPS = 归母净利润 / 总股本。
- 营收缺失不可臆造，表格内写 `暂无`。
- 评级或目标价缺失时表格内写 `暂无`。
- 目标价必须保留原始币种或明确换算币种。
- 同一机构多条记录只保留最新一条。
- 自行计算项必须在内部数据核验记录中记录公式，并在表格下方写简短口径说明。
- 一致预期表格中每一行关键数据必须有来源角标；若同一年预测或同一机构评级来自同一来源页，可共用同一来源编号。
- 无覆盖说明不需要强行角标；若引用外部页面证明无覆盖，则该页面必须可打开并进入“信息来源”。
